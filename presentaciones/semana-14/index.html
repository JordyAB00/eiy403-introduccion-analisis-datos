¿Qué es machine learning?

Machine learning es un método de análisis de datos que automatiza la construcción de modelos. Es una rama de la inteligencia artificial basada en la idea de que los sistemas pueden aprender de datos, identificar patrones y tomar decisiones con mínima intervención humana.

Programación tradicional

Datos
+
Reglas
→
Resultados

Ejemplo:

Para clasificar emails como spam:

  • Programador escribe reglas: "Si contiene 'ganador', marca como spam"
  • El programa aplica estas reglas
  • Muy limitado y rígido

Machine learning

Datos
+
Resultados
→
Reglas

Ejemplo:

Para clasificar emails como spam:

  • Damos miles de emails etiquetados
  • El algoritmo descubre patrones
  • Aprende reglas automáticamente
  • Se adapta a nuevos tipos de spam

Diferencia clave: En programación tradicional, nosotros le decimos al computador QUÉ hacer. En machine learning, le mostramos ejemplos y el computador aprende QUÉ hacer por sí mismo.

Tipos de machine learning

📚

Aprendizaje supervisado

Aprendemos de ejemplos etiquetados. Tenemos datos con respuestas correctas y el modelo aprende a predecir.

Ejemplos:

  • Predecir precio de casas
  • Clasificar emails (spam/no spam)
  • Diagnóstico médico
  • Nuestro proyecto: predecir resultados de partidos
🔍

Aprendizaje no supervisado

Buscamos patrones ocultos sin etiquetas. El modelo explora y agrupa datos similares.

Ejemplos:

  • Segmentación de clientes
  • Detección de anomalías
  • Sistemas de recomendación
  • Compresión de datos
🎮

Aprendizaje por refuerzo

Aprendemos por prueba y error. El modelo recibe recompensas o castigos por sus acciones.

Ejemplos:

  • Juegos (AlphaGo, ajedrez)
  • Robots autónomos
  • Vehículos autónomos
  • Trading financiero
Hoy nos enfocaremos en aprendizaje supervisado, el tipo más común y útil para problemas con datos históricos

Regresión (predecir números)

¿Qué predice? Valores continuos (números)

Ejemplos:

  • Regresión lineal

  • Random Forest

Ejemplos:

  • Paciente tiene enfermedad o no

Algoritmos comunes:

  • Regresión logística
  • Árboles de decisión

  • Random Forest

El proceso completo de machine learning

1. Definir el problema
↓
2. Recolectar datos
↓
3. Explorar y limpiar datos
↓
4. Ingeniería de características
↓
5. Entrenar modelo
↓
6. Evaluar modelo
↓
7. Hacer predicciones

Pasos 1-4: Preparación (70% del tiempo)

  • Definir problema: ¿Qué queremos predecir? ¿Por qué?
  • Recolectar datos: Encontrar o generar datos relevantes
  • Explorar y limpiar: EDA, manejar valores faltantes, outliers
  • Ingeniería: Crear variables útiles para el modelo

Pasos 5-7: Modelado (30% del tiempo)

  • Entrenar: Dividir datos, seleccionar algoritmo, ajustar
  • Evaluar: Medir qué tan bien funciona el modelo
  • Predecir: Usar el modelo en casos nuevos

Error común: Muchos principiantes piensan que ML es solo "entrenar un modelo". En realidad, la mayor parte del trabajo es preparar bien los datos. Garbage in, garbage out (basura entra, basura sale).

Ingeniería de características (features)

"Los datos crudos rara vez están listos para ML. Necesitamos transformarlos en características que el modelo pueda entender"

¿Qué son las características (features)?

Las características son las variables que el modelo usa para hacer predicciones. Son como las "pistas" que le damos al modelo.

Características crudas (malas)

Ejemplo: Predecir precio de casa

  • Dirección: "123 Calle Principal"
  • Fecha construcción: "15/03/1985"
  • Color: "Azul"

Problema: El modelo no puede aprender de texto o fechas directamente

Características procesadas (buenas)

Mismo ejemplo procesado:

  • Metros cuadrados: 150
  • Distancia al centro: 5.2 km
  • Antigüedad: 40 años
  • Número de habitaciones: 3

Mejor: Números que el modelo puede procesar

División entrenamiento y prueba

Pregunta crucial: Si usamos TODOS los datos para entrenar, ¿cómo sabemos si el modelo realmente funciona en casos nuevos?

La solución: dividir los datos

↓
Entrenamiento (80%)

El modelo APRENDE de estos datos

Prueba (20%)

Evaluamos el modelo aquí

Conjunto de entrenamiento

  • Representa el 70-80% de los datos
  • El modelo ve estos datos durante entrenamiento
  • Aprende patrones y relaciones
  • Ajusta sus parámetros internos

Analogía: Como estudiar con ejercicios antes del examen

Conjunto de prueba

  • Representa el 20-30% de los datos
  • El modelo NUNCA ve estos datos durante entrenamiento

  • Simula casos completamente nuevos

Overfitting y underfitting

El desafío principal en ML: crear un modelo que aprenda patrones generales, no que memorice los datos específicos

Underfitting

📉

Problema: El modelo es demasiado simple

  • No captura patrones importantes
  • Bajo desempeño en entrenamiento
  • Bajo desempeño en prueba

Analogía: Estudiante que no estudió lo suficiente

Buen ajuste

✓

Ideal: El modelo aprende patrones generales

  • Captura relaciones importantes
  • Buen desempeño en entrenamiento
  • Buen desempeño en prueba

Analogía: Estudiante que entendió los conceptos

Overfitting

📈

Problema: El modelo es demasiado complejo

  • Memoriza los datos de entrenamiento
  • Excelente desempeño en entrenamiento
  • Bajo desempeño en prueba

Analogía: Estudiante que memorizó ejercicios sin entender

Ejemplo visual

Situación Entrenamiento Prueba Diagnóstico Modelo A 60% accuracy 58% accuracy Underfitting Modelo B 85% accuracy 82% accuracy Buen ajuste Modelo C 98% accuracy 65% accuracy Overfitting

Lección clave: Queremos que el desempeño en entrenamiento y prueba sean similares. Una gran diferencia indica overfitting.

¿Cómo evaluamos un modelo?

Métricas para clasificación

Accuracy (precisión general)

Accuracy = (Predicciones correctas) / (Total de predicciones)

Ejemplo: Si predecimos 100 partidos y acertamos 70, accuracy = 70%

Matriz de confusión

Muestra dónde el modelo acierta y dónde se equivoca:

Predicho: Victoria Local Predicho: Empate Predicho: Victoria Visitante
Real: Victoria Local 15 3 2
Real: Empate 2 8 2
Real: Victoria Visitante 1 2 6

En verde: Predicciones correctas

En rojo: Errores del modelo

¿Qué es un buen accuracy?

Depende del problema:

  • Diagnóstico médico: Necesitamos 95%+ (vidas en juego)
  • Detección de spam: 90%+ es aceptable
  • Predicción financiera: 60-70% puede ser rentable
  • Predicción deportiva: 55-65% es bueno (hay mucho azar)

Comparar con el azar:

En nuestro proyecto, si adivináramos al azar entre 3 opciones (victoria local, empate, victoria visitante), tendríamos 33% de accuracy. Por lo tanto, cualquier modelo debe superar este 33% base.

En fútbol: Debido a la alta variabilidad e imprevisibilidad del deporte, un accuracy de 60-70% se considera excelente.

Algoritmos de ML comunes

Regresión logística

Uso: Clasificación (a pesar del nombre)

Cómo funciona: Calcula probabilidades de cada clase usando una función matemática

Ventajas:

  • Simple y rápido
  • Interpretable
  • Funciona bien con pocas muestras

Usaremos este en nuestro proyecto

Árboles de decisión

Uso: Clasificación y regresión

Cómo funciona: Hace preguntas sucesivas como un diagrama de flujo

Ventajas:

  • Muy interpretable
  • No necesita normalización
  • Maneja datos categóricos bien

Random Forest

Uso: Clasificación y regresión

Cómo funciona: Combina muchos árboles de decisión

Ventajas:

  • Muy preciso
  • Robusto contra overfitting
  • Maneja muchas variables

Redes neuronales

Uso: Problemas complejos (imágenes, texto)

Cómo funciona: Imita neuronas del cerebro

Ventajas:

  • Muy poderoso para datos complejos
  • Aprende características automáticamente

Desventajas:

  • Necesita MUCHOS datos
  • Costoso computacionalmente
  • Difícil de interpretar

¿Cómo elegir? Depende de:

  • Cantidad de datos disponibles
  • Complejidad del problema
  • Necesidad de interpretabilidad
  • Recursos computacionales
  • Tiempo disponible

Regla práctica: Empieza simple (regresión logística) y complejiza solo si es necesario

Limitaciones del machine learning

ML no es magia. Tiene limitaciones importantes que debemos entender.

Limitaciones técnicas

1. Calidad de datos

  • Basura entra, basura sale
  • Datos sesgados producen modelos sesgados
  • Datos insuficientes limitan el aprendizaje

2. Correlación vs causalidad

  • ML encuentra correlaciones, no causas
  • Puede encontrar patrones espurios
  • Ejemplo: helado y ahogamientos (correlacionan por el verano)

3. Extrapolación

  • Modelos fallan fuera del rango de entrenamiento
  • No pueden predecir situaciones totalmente nuevas
  • Ejemplo: modelo entrenado pre-pandemia falla en pandemia

En nuestro proyecto de fútbol

Variables que NO podemos capturar:

  • Estado físico: Lesiones de jugadores clave
  • Motivación: Presión psicológica del momento
  • Táctica específica: Plan de juego del entrenador
  • Clima: Lluvia, calor extremo el día del partido
  • Errores arbitrales: Decisiones polémicas que cambian el partido
  • Suerte: Gol de rebote en el minuto 90

Por eso:

  • Un accuracy de 60% en fútbol es realista
  • Nunca esperemos 95%+ de precisión
  • El modelo da probabilidades, no certezas
  • Siempre habrá sorpresas e incertidumbre

Ética y responsabilidad en ML

Con gran poder viene gran responsabilidad. ML puede tener impactos significativos en la sociedad.

Problemas éticos comunes

1. Sesgo algorítmico

  • Modelos aprenden sesgos de los datos históricos
  • Ejemplo: IA de contratación discrimina contra mujeres
  • Perpetúa desigualdades existentes

2. Privacidad

  • Modelos pueden revelar información sensible
  • Riesgo de identificar individuos en datos "anónimos"
  • Uso no autorizado de datos personales

3. Transparencia

  • Modelos complejos son "cajas negras"
  • Difícil explicar decisiones automáticas
  • Problema en medicina, justicia, finanzas

4. Responsabilidad

  • ¿Quién es responsable si el modelo falla?
  • ¿El programador? ¿La empresa? ¿El algoritmo?
  • Importante en vehículos autónomos, medicina

Principios para ML responsable

1. Fairness (justicia)

  • Evaluar sesgos en datos y modelos
  • Asegurar equidad entre grupos
  • Diversidad en equipos de desarrollo

2. Accountability (rendición de cuentas)

  • Documentar decisiones de diseño
  • Auditorías externas de modelos
  • Mecanismos de apelación

3. Transparency (transparencia)

  • Explicar cómo funciona el modelo
  • Comunicar limitaciones
  • Mostrar datos de entrenamiento

4. Privacy (privacidad)

  • Minimizar datos personales
  • Consentimiento informado
  • Técnicas de anonimización

En nuestro proyecto:

Como es predicción deportiva, el impacto ético es bajo. Pero es importante practicar estos principios:

  • Ser honestos sobre limitaciones
  • No sobrevender el modelo
  • Explicar cómo funciona
  • Reconocer incertidumbre
