Limitaciones de estructuras simples

Problema 1: datos heterog√©neos

Un experimento qu√≠mico puede tener: nombre (texto), temperatura (n√∫mero), resultados (vector), y condiciones (data frame)

Problema 2: organizaci√≥n l√≥gica

Necesitamos agrupar informaci√≥n relacionada de manera que tenga sentido conceptualmente

Problema 3: an√°lisis complejos

Los an√°lisis estad√≠sticos devuelven m√∫ltiples resultados (coeficientes, p-valores, gr√°ficos, etc.)

  • Jerarqu√≠a l√≥gica de datos

Flexibilidad total

Listas en R: conceptos fundamentales

Definici√≥n

Una lista es una estructura de datos que puede contener elementos de diferentes tipos y estructuras. Es como un contenedor flexible que puede almacenar cualquier cosa.

Creaci√≥n de listas simples

# Lista simple mi_lista <- list(42, "texto", TRUE, 3.14) # Ver la estructura str(mi_lista) # List of 4 # $ : num 42 # $ : chr "texto" # $ : logi TRUE # $ : num 3.14 # Ver contenido mi_lista # [[1]] # [1] 42 # # [[2]] # [1] "texto" # # [[3]] # [1] TRUE # # [[4]] # [1] 3.14

Lista con vectores

# Lista conteniendo vectores datos_lab <- list( c(25.1, 25.3, 24.9), # temperaturas c(7.2, 7.1, 7.3), # pH c("A", "B", "A") # catalizador ) # Acceder al primer elemento (vector de temperaturas) datos_lab[[1]] # [1] 25.1 25.3 24.9

Acceso a elementos

Tres formas de acceder:

mi_lista <- list(10, 20, 30) # 1. Con [[ ]] - devuelve el elemento mi_lista[[1]] # [1] 10 # 2. Con [ ] - devuelve una sublista mi_lista[1] # [[1]] # [1] 10 # 3. Por posici√≥n m√∫ltiple mi_lista[c(1, 3)] # [[1]] # [1] 10 # # [[2]] # [1] 30

Diferencia importante:

  • [[1]] ‚Üí extrae el elemento directamente
  • [1] ‚Üí extrae una sublista con ese elemento

Listas anidadas

# Lista dentro de otra lista experimentos <- list( exp1 = list(temp = 25, pH = 7.0), exp2 = list(temp = 30, pH = 6.5) ) # Acceso anidado experimentos[[1]][[1]] # 25 experimentos[[2]][[2]] # 6.5

Listas nombradas: el equivalente a diccionarios

¬øQu√© son?

Las listas nombradas son listas donde cada elemento tiene un nombre asociado. Esto hace el c√≥digo m√°s legible y es el equivalente directo a los diccionarios en Python.

Creaci√≥n

# Sintaxis 1: al crear la lista experimento <- list( nombre = "S√≠ntesis de aspirina", fecha = "2025-10-08", temperatura = 25.5, pH = 7.2, concentraciones = c(0.1, 0.2, 0.15) ) # Sintaxis 2: asignar nombres despu√©s valores <- list(100, 200, 300) names(valores) <- c("minimo", "medio", "maximo") # Ver nombres names(experimento) # [1] "nombre" "fecha" # [3] "temperatura" "pH" # [5] "concentraciones"

Ventajas

  • Legibilidad: nombres descriptivos vs. posiciones num√©ricas
  • Mantenimiento: f√°cil agregar/quitar elementos
  • Autoexplicativo: el c√≥digo se documenta solo

Acceso a elementos nombrados

# Tres formas de acceder por nombre: # 1. Con $ (m√°s com√∫n y limpio) experimento$temperatura # [1] 25.5 # 2. Con [["nombre"]] experimento[["pH"]] # [1] 7.2 # 3. Con ["nombre"] (devuelve sublista) experimento["nombre"] # $nombre # [1] "S√≠ntesis de aspirina"

Modificaci√≥n

# Cambiar un valor existente experimento$temperatura <- 30.0 # Agregar un nuevo elemento experimento$presion <- 1.2 # Eliminar un elemento experimento$presion <- NULL # Ver estructura actualizada str(experimento)

Introducci√≥n a clases en R (sistema S3)

¬øQu√© son las clases?

En R, las clases permiten crear tipos de objetos personalizados con comportamientos espec√≠ficos. El sistema S3 es el m√°s simple y com√∫n.

Concepto b√°sico

Una clase S3 es simplemente una lista con un atributo "class". Esto permite que funciones gen√©ricas (como print(), summary(), plot()) se comporten de manera diferente seg√∫n el tipo de objeto.

Crear un objeto con clase
