¬øPor qu√© visualizar datos?

El poder de la visualizaci√≥n

Procesamiento visual

  • El cerebro procesa im√°genes 60,000 veces m√°s r√°pido que texto
  • 90% de la informaci√≥n transmitida al cerebro es visual
  • Recordamos 80% de lo que vemos vs 20% de lo que leemos

Identificaci√≥n de patrones

  • Detecta tendencias instant√°neamente
  • Identifica valores at√≠picos (outliers)
  • Revela relaciones entre variables
  • Muestra distribuciones de datos

Comunicaci√≥n efectiva

  • Simplifica datos complejos
  • Cuenta historias con datos
  • Persuade con evidencia visual
  • Facilita toma de decisiones

Ejemplo: el cuarteto de Anscombe

Cuatro conjuntos de datos con las mismas estad√≠sticas pero patrones visuales muy diferentes:

Todos tienen:

  • Media de X: 9.0
  • Media de Y: 7.5
  • Desviaci√≥n est√°ndar de X: 3.32
  • Desviaci√≥n est√°ndar de Y: 2.03
  • Correlaci√≥n: 0.816
  • Ecuaci√≥n de regresi√≥n: y = 3 + 0.5x

Pero al graficarlos...

  • Conjunto 1: Relaci√≥n lineal perfecta
  • Conjunto 2: Relaci√≥n no lineal (curva)
  • Conjunto 3: Relaci√≥n lineal con outlier
  • Conjunto 4: Sin relaci√≥n, solo un punto extremo
"Las estad√≠sticas resumen pueden mentir. Los gr√°ficos revelan la verdad"

Principios de dise√±o gr√°fico

Principio 1: Claridad

  • Un mensaje principal: cada gr√°fico debe comunicar UNA idea
  • T√≠tulo descriptivo: que explique qu√© muestra el gr√°fico
  • Ejes etiquetados: siempre con unidades
  • Leyenda clara: cuando hay m√∫ltiples series

Principio 2: Simplicidad

  • Menos es m√°s: elimina elementos innecesarios
  • Sin chartjunk: nada de 3D, sombras, efectos
  • Ratio tinta-datos: maximiza informaci√≥n, minimiza decoraci√≥n
  • Espacio en blanco: usa m√°rgenes apropiados

Principio 3: Honestidad

  • Eje Y en cero: para gr√°ficos de barras
  • Escalas apropiadas: no exagerar diferencias
  • Datos completos: no ocultar informaci√≥n relevante
  • Contexto necesario: incluye tama√±o de muestra, errores

Principio 4: Consistencia

  • Colores uniformes: misma variable = mismo color
  • Tipograf√≠a coherente: una o dos fuentes m√°ximo
  • Escalas comparables: entre gr√°ficos relacionados
  • Estilo √∫nico: en toda la presentaci√≥n/documento

Principio 5: Accesibilidad

  • Colores distinguibles: para personas con daltonismo
  • Tama√±o de texto: legible a distancia
  • Contraste suficiente: fondo vs elementos
  • Alternativas: patrones adem√°s de colores

Principio 6: Contexto

  • Audiencia: adapta complejidad al p√∫blico
  • Prop√≥sito: exploraci√≥n vs presentaci√≥n
  • Medio: impreso, pantalla, proyecci√≥n
  • Comparaciones: incluye referencias o benchmarks

Malas pr√°cticas: chartjunk y efectos 3D

¬øQu√© es chartjunk?

Elementos visuales que no agregan informaci√≥n, solo decoraci√≥n que distrae o confunde.

Ejemplos de chartjunk:

  • Efectos 3D: Distorsionan la percepci√≥n de valores
  • Gradientes excesivos: En barras o √°reas
  • Sombras y brillos: No agregan informaci√≥n
  • Fondos con im√°genes: Dificultan lectura
  • Grillas excesivas: Demasiadas l√≠neas de referencia
  • Bordes gruesos: Resaltan el marco, no los datos
  • Iconos decorativos: Pictogramas innecesarios

El problema del 3D:

Los gr√°ficos 3D son casi siempre una mala idea porque:

  • Distorsionan los valores reales por perspectiva
  • Ocultan datos (barras traseras)
  • Dificultan comparaciones precisas
  • Agregan complejidad sin beneficio

Ejemplo cl√°sico: Gr√°fico de torta 3D

Las secciones del frente parecen m√°s grandes que las del fondo, incluso con el mismo porcentaje. Es el peor tipo de gr√°fico posible.

C√≥mo evitarlo en R

# ‚ùå MAL: Gr√°fico con chartjunk datos <- c(20, 35, 25, 20) pie(datos, col = rainbow(4), # Colores chillones border = "white", # Sin prop√≥sito density = 10, # L√≠neas que distraen angle = 45) # M√°s distracci√≥n # ‚úì BIEN: Gr√°fico limpio barplot(datos, col = "#3498db", # Un color s√≥lido border = NA, # Sin bordes main = "Distribuci√≥n de datos", ylab = "Frecuencia") grid(nx = NA, ny = NULL) # Solo l√≠neas horizontales

Checklist de un buen gr√°fico

‚úì Contenido

  • Tiene un mensaje claro y √∫nico
  • Muestra datos honestos sin distorsi√≥n
  • Incluye todos los datos relevantes

‚úì Dise√±o

  • T√≠tulo descriptivo y conciso
  • Ejes etiquetados con unidades
  • Leyenda clara (si es necesaria)
  • Escala apropiada (eje Y en cero para barras)
  • Sin elementos innecesarios (chartjunk)

‚úì Color y est√©tica

  • Paleta de colores coherente y profesional
  • Accesible para personas con daltonismo
  • Contraste suficiente
  • No m√°s de 5-7 colores diferentes
  • Colores con significado (no arbitrarios)

‚úì Legibilidad

  • Texto de tama√±o legible
  • Fuentes simples y profesionales
  • Etiquetas no superpuestas
  • Espacio en blanco apropiado

Errores que NUNCA debes cometer

‚ùå Prohibido:

  • Gr√°ficos de torta (usa barras)
  • Efectos 3D innecesarios
  • Eje Y que no empieza en cero (en barras)

  • Demasiados colores saturados
  • M√°s de 10 categor√≠as en un gr√°fico
  • Texto ilegible o rotado 90¬∞
  • Leyendas con demasiadas entradas
  • Ocultar datos para "embellecer"
