¬øQu√© es el an√°lisis exploratorio de datos?

"El EDA es el proceso de investigar datos para descubrir lo desconocido, cuestionar lo asumido, y revelar patrones ocultos antes de aplicar modelos formales"

Objetivos Fundamentales

  • Comprensi√≥n: Entender qu√© dicen realmente los datos
  • Validaci√≥n: Verificar la calidad e integridad
  • Descubrimiento: Encontrar patrones inesperados
  • Preparaci√≥n: Limpiar y preparar para an√°lisis
  • Comunicaci√≥n: Visualizar hallazgos clave

Filosof√≠a del EDA

  • Los datos tienen historias que contar
  • Nuestras suposiciones pueden estar equivocadas
  • La visualizaci√≥n revela m√°s que los n√∫meros
  • La calidad de los datos determina la calidad del an√°lisis

Lo que NO es EDA

  • NO es solo hacer gr√°ficos bonitos
  • NO es calcular estad√≠sticas b√°sicas
  • NO es un paso opcional
  • NO es algo que se hace una sola vez

Importancia Cr√≠tica

  • 80% del tiempo en ciencia de datos se gasta en EDA
  • Previene errores costosos en el an√°lisis
  • Genera hip√≥tesis para investigaci√≥n futura
  • Mejora la comunicaci√≥n de resultados

Estrategias para Datos Faltantes

Eliminar (Listwise deletion)

Cu√°ndo usarlo:

  • < 5% de datos faltantes
  • Datos faltantes completamente aleatorios
  • Dataset muy grande

Imputar con medidas centrales

Imputaci√≥n avanzada

  • Regresi√≥n lineal
  • K-nearest neighbors
  • M√©todos basados en modelos

Estrategias para duplicados

Identificaci√≥n

  • Duplicados exactos: Todas las columnas iguales

  • Duplicados conceptuales: Misma entidad, diferentes representaciones

Decisiones de acci√≥n

  • Eliminar autom√°ticamente: Duplicados exactos
  • Revisar manualmente: Duplicados parciales
  • Crear reglas: Para duplicados conceptuales
  • Conservar el m√°s reciente: Cuando hay timestamps

CUIDADO: Los outliers pueden ser:

  • Errores de captura (eliminar)
  • Fen√≥menos raros pero reales (conservar)

Medidas de tendencia central: ¬øCu√°l elegir?

Media Aritm√©tica ($\bar{x}$)

$$\bar{x} = \frac{\sum_{i=1}^{n} x_i}{n}$$

Usar cuando:

  • Distribuci√≥n sim√©trica
  • Sin valores at√≠picos extremos
  • Datos de escala intervalo o raz√≥n
  • Se necesita precisi√≥n matem√°tica

Evitar cuando:

  • Distribuci√≥n muy asim√©trica
  • Presencia de outliers
  • Datos ordinales
  • Datos con valores extremos influyentes

Ejemplo problem√°tico: Ingresos familiares con algunos millonarios en la muestra ‚Üí la media ser√° enga√±osa

Mediana (Me) y Moda (Mo)

Mediana - El valor del medio:

  • Robusta ante outliers
  • Apropiada para datos ordinales
  • Mejor para distribuciones asim√©tricas
  • Interpretaci√≥n clara: 50% por encima/debajo

Moda - El valor m√°s frecuente:

  • √önica medida para datos nominales
  • Identifica valores t√≠picos/populares
  • Puede tener m√∫ltiples valores
  • Puede no existir o ser poco informativa

Regla pr√°ctica:

  • Sim√©trica: Media ‚âà Mediana ‚âà Moda
  • Asim√©trica positiva: Media > Mediana > Moda
  • Asim√©trica negativa: Media < Mediana < Moda

Medidas de tendencia no central

Cuartiles y Percentiles

$$Q_1 = P_{25}, \quad Q_2 = P_{50} = Me, \quad Q_3 = P_{75}$$

Interpretaci√≥n:

  • Q1: 25% de datos por debajo
  • Q2: Mediana (50%)
  • Q3: 75% de datos por debajo
  • P90: 90% de datos por debajo

Rango Intercuart√≠lico (IQR)

$$IQR = Q_3 - Q_1$$

Ventajas del IQR:

  • Robusto ante outliers
  • Describe el 50% central de los datos
  • Base para detectar outliers
  • √ötil para comparar dispersi√≥n

Medidas de dispersi√≥n: ¬øQu√© tan dispersos est√°n los datos?

Varianza y Desviaci√≥n Est√°ndar

$$\text{Varianza poblacional: } \sigma^2 = \frac{\sum(x_i - \mu)^2}{N}$$ $$\text{Varianza muestral: } s^2 = \frac{\sum(x_i - \bar{x})^2}{n-1}$$ $$\text{Desviaci√≥n est√°ndar: } s = \sqrt{s^2}$$

¬øPor qu√© n-1 en la muestra?

  • Correcci√≥n de Bessel
  • Estimador insesgado de œÉ¬≤
  • Perdemos 1 grado de libertad al estimar $\bar{x}$

Interpretaci√≥n de la Desviaci√≥n Est√°ndar

Regla Emp√≠rica (distribuci√≥n normal):

  • 68% de datos en $\bar{x} \pm 1s$
  • 95% de datos en $\bar{x} \pm 2s$
  • 99.7% de datos en $\bar{x} \pm 3s$

Coeficiente de Variaci√≥n

$$CV = \frac{s}{\bar{x}} \times 100\%$$

Interpretaci√≥n del CV:

  • CV < 15%: Baja variabilidad
  • 15% ‚â§ CV ‚â§ 30%: Variabilidad moderada
  • CV > 30%: Alta variabilidad

¬øCu√°ndo usar CV?

  • Comparar variabilidad entre datasets con diferentes unidades
  • Comparar variabilidad cuando las medias son muy diferentes
  • Evaluar precisi√≥n de mediciones

Limitaci√≥n: CV no es apropiado cuando la media es cercana a cero o puede ser negativa

Otras Medidas de Dispersi√≥n

  • Rango: M√°ximo - M√≠nimo (muy sensible a outliers)
  • MAD: Mediana de las desviaciones absolutas
  • Rango intercuart√≠lico: Q3 - Q1 (robusto)

# Limpieza iterativa de outliers extremos
datos_limpios <- datos
outliers_encontrados <- TRUE
iteracion <- 0
max_iteraciones <- 5

while (outliers_encontrados && iteracion < max_iteraciones) {
    # Calcular l√≠mites IQR
    Q1 <- quantile(datos_limpios$concentracion, 0.25, na.rm = TRUE)
    Q3 <- quantile(datos_limpios$concentracion, 0.75, na.rm = TRUE)
    IQR_val <- Q3 - Q1
    
    # Detectar outliers extremos
    limite_inf <- Q1 - 3 * IQR_val  # M√°s estricto (3 x IQR)
    limite_sup <- Q3 + 3 * IQR_val
    
    outliers <- which(datos_limpios$concentracion < limite_inf | 
                      datos_limpios$concentracion > limite_sup)
    
    if (length(outliers) == 0) {
        outliers_encontrados <- FALSE
        cat("No se encontraron m√°s outliers extremos\n")
    } else {
        # Remover outliers
        datos_limpios <- datos_limpios[-outliers, ]
        iteracion <- iteracion + 1
        cat("Iteraci√≥n", iteracion, "- Removidos", length(outliers), "outliers\n")
    }
}

cat("Proceso completado en", iteracion, "iteraciones")
cat("Datos finales:", nrow(datos_limpios), "de", nrow(datos), "originales")

    # Verificar que el archivo existe
    if (file.exists(archivo)) {
        
        # Manejo de errores con tryCatch
        datos_archivo <- tryCatch({
            read.csv(archivo, stringsAsFactors = FALSE)
        }, error = function(e) {
            cat("Error leyendo", archivo, ":", e$message, "\n")
            return(NULL)
        })
        
        # Continuar solo si la lectura fue exitosa
        if (!is.null(datos_archivo)) {
            # Validar estructura esperada
            if (all(c("muestra_id", "ph", "concentracion") %in% names(datos_archivo))) {
                # Procesar datos...
                cat("Procesado exitosamente:", archivo, "\n")
            } else {
                cat("Columnas faltantes en", archivo, "\n")
            }
        }
    } else {
        cat("Archivo no encontrado:", archivo, "\n")
    }
}
